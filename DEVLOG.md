## Phase 6: MVP 範圍鎖定 + Claude Design 提示詞 (2026-06-26)

### 背景

確認 MVP 設計決策後，使用者改變方向：不直接寫程式，改要一份**完整提示詞給 Claude Design** 做設計。
並追加視覺要求：白底、介面乾淨像 Apple 官網、所有可互動元素 hover 都要 highlight。

### 確認的設計決策

- MVP 情境：**警勤 + 軍事**（智慧城市排除）。
- 人數：**預設級距按鈕**（1/2/4/6/8/12/16/24）。
- 場地：**預設級距**（小<36 / 中36–81 / 大81–300 / 超大>300 ㎡）。
- 武器費用：**分級假價**（手槍8萬…飛彈30萬…載具50萬）。

### 做了什麼

1. 短暫開始寫純前端骨架（`assets/js/data.js`）→ 使用者喊停，改要提示詞 → **已移除該半成品**。
2. 產出 `doc/claude-design-提示詞.md`：含可直接複製的提示詞區塊 + 內容資料附錄
   （情境/武器分級假價/人數場地級距/定位方式約束/課程模板）。
3. 提示詞涵蓋：Apple 風白底乾淨、hover highlight、兩欄版面、約束式自動禁用、
   即時更新、左側動態示意圖、RWD、純前端技術要求。

### 下次接續點

- 使用者拿提示詞去 Claude Design 產生設計/前端。
- 待回饋：拿到設計稿後，校正假價為真值、串接 Excel 真實 BOM、補品牌配色（待官網截圖）。
