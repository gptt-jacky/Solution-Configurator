# 需求整理 — 產品配置器會議

## 1. 產品定位（一句話）

**模擬訓練解決方案配置器**——給**業務／行銷**用的工具，協助面對政府／軍方客戶時，
快速把「客戶模糊的需求」轉成「具體的解決方案 + 價格 + 視覺示意圖」。

> 不是消費性產品；軍事/政府案的業務角色非常重要，這是輔助業務溝通的工具。

## 2. 核心痛點（要解決的問題）

- 新客戶**不知道自己要什麼**，只給得出模糊資訊：
  例如「年齡 8–20 歲、想練步槍/手槍、場地可能 8×8m 或 30×30m」。
- 業務以前**接不起來**這種前期諮詢（需求講不清、技術太複雜）。
- 配置器要讓客戶/業務**很快知道**：要這麼大場地要花多少錢、要這麼多功能要多少錢、大概長什麼樣。

## 3. 關鍵設計原則

| 原則 | 說明 |
|------|------|
| **隱藏技術術語** | inside-out / outside-in / 紅外線等，業務和客戶都不懂、也不該懂。用白話：**場地越大 → 推某方案；要定位的東西越多 → 推某方案** |
| **約束式篩選（核心難點 ★）** | 選了某個選項，**自動消除/禁用不相容的選項**。多種定位系統各有限制、彼此衝突、各有優缺點，這是最難的部分 |
| **引導式流程** | 把複雜資訊做成流程，讓客戶快速知道自己需要什麼 |
| **輸出推薦方案** | 客戶勾完需求 → 列出**最優方案**（不是讓他硬選技術），附**價格** |
| **視覺化** | 技術配置限制、部件廠商都視覺化；左側示意圖呈現大概樣貌 |

## 4. 客戶在意點（因人而異，需可彈性側重）

- **情境/沉浸感**：有些客戶只在意情境 → 頭盔畫面要好、顏值要高
- **武器種類**：種類多 → 著重武器選擇（每把武器約抓 **8–10 萬**開發+製作費）
- **人數**：影響大（如 LBE mode、人數多價格就上去）
- **場域大小**：基本成本驅動

> 成本驅動主要是：人數、設備量、武器、場地大小、定位精度需求。

## 5. 資料來源

- **核心是 SUT / SOT**（公司目前做得最完整的產品）。
- 其他客戶以**專案形式**選任意定位方案，透過此配置器選擇。
- 資料來自 `XR解決方案成本試算.xlsx`（即 DATA_MODEL.md 整理的那張定位方式總表）。

## 6. 範圍與態度（MVP）

- **價格先做假的/粗估即可**，不需精確（之後再校正）。

- 選配項目（SR 等 option）可列入，給客戶選。
