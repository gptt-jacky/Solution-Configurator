# Claude Design 提示詞 v4 — 模擬訓練解決方案配置器

> 依「產品配置器5」會議回饋優化。整體沿用 v3，本版主要為細節打磨。
> 使用方式：在現有 v3 成品上續做——開頭加「請以我現有的版本為基礎，套用以下優化」，
> 再貼「===提示詞開始===」到「===提示詞結束===」整段 + 下方附錄；
> 並確認 `槍枝照片/` 已上傳、**貼上橫向 logo 圖**。

## 本版改動摘要（相對 v3）

1. ⚠️ **Hover highlight 務必做（已第三次提醒）**：所有可互動元素滑鼠移過去要**微微上浮 + highlight**，這是最高優先、絕不可再漏。
2. **武器轉盤微調**：中央（focus）武器**放大到 130%**、兩側武器縮小；**武器之間間距加大約 30%**（目前過渡太近、動態看起來怪）。
3. **快速套用課程列**：滑鼠移到該列上時，**滾輪可橫向捲動**課程（hover 在框內向下滾＝課程往左移），不必只靠點箭頭。
4. **快速套用按鈕加按壓特效**：按住時變藍、放開回白（要有明確的按下互動回饋）。
5. **定位方案的圖片移到「方案內容(BOM)」下方那一區**呈現，不要放在右側選擇器裡。
6. **頂部使用橫向 Logo**（全球動力科技，使用者會提供圖檔）。
7. **配色維持：藍色 UI（#0071e3）+ 紅色 Logo**（logo 緋紅 ≈ #D6173E，僅作為頂部標誌，UI 主色仍是藍）。
8. 手槍維持**多重區分（電動/瓦斯）**。
9. **拿掉「應用情境」上方的藍點**（裝飾性小圓點移除）。
10. 其餘 v3 規格不變（左視覺凍結＋右選單捲動、價格頂部全寬凍結、Apple 大圓角＋SF Pro、滅火器限 1–4 人…）。

# 角色

你是資深 **產品設計師 + 前端工程師**，擅長 Apple 官網等級的極簡、乾淨、高質感互動網頁。
請在我現有版本基礎上，繼續打磨這個**互動式產品配置器網站**（汽車線上選配風格），純前端、可直接用瀏覽器開。

# 產品背景

軍警消 XR 模擬訓練公司（全球動力科技 / BigxReality）的**模擬訓練解決方案配置器**，給**業務／行銷**用。
把客戶模糊需求引導成**具體方案 + 即時報價 + 場域示意圖**，**武器/裝備是賣點主角**。

# 最高指導原則（務必嚴格遵守）

1. ⚠️ **所有可互動元素，滑鼠 hover 一律要有 highlight**：微微上浮（translateY -1~2px）+ 輕陰影 + 底色/邊框變化，過渡 150–200ms。**此項已被要求三次仍未落實，本版務必確實做到每一個按鈕/卡片/選項/清單項。**
2. **視覺像 Apple 官網**：極簡、大量留白、層級清楚、內容優於裝飾。
3. **背景純白** `#ffffff`，次級區塊 `#f5f5f7`。
4. **UI 主色統一藍** `#0071e3`（選中態、主按鈕、價格一律藍）；情境切換不換色。
   **頂部 Logo 為紅色橫向標誌**（使用者提供，緋紅 ≈ #D6173E）；logo 是唯一的紅，其餘 UI 維持藍。
5. **圓角用明顯的 Apple 連續曲率（squircle）大圓角**，全站統一（v3 已達標，維持）。
6. **字體比照 Apple**：`-apple-system, "SF Pro Display", "SF Pro Text", "PingFang TC", "Noto Sans TC", sans-serif`。
7. **即時反應**：任一選擇變動 → 轉盤、場景、價格、BOM 立即更新。
8. **約束式篩選**：選場地後不相容定位方式自動變灰禁用並標原因。
9. **隱藏技術術語**：只講白話（場地大小、能不能施工）。

# 版面結構（沿用 v3）

- **頂部全寬價格列（凍結）**：左＝**橫向 Logo**（全球動力科技）+「方案配置器」；右＝預估價格大字（藍）+「匯出報價」（藍）+「加入清單」+「重置」。不論情境一律藍。
- **左欄（凍結，永遠在）**：武器轉盤（主角）+ 場景示意圖。捲動時左欄不動。
- **右欄（捲動）**：快速套用課程 → ① 應用情境（三選一）→ ② 武器與裝備（在上）→ ③ 場地大小 → ④ 人數 → ⑤ 定位方式（自動禁用、低調）。
- 第一屏右欄要同時看到武器與場地。
- 下方：方案內容（BOM）+ 規格摘要 +（本版）定位方案圖片區。

# 本版重點細節

### 武器轉盤（微調）
- 已選武器以去背產品照做橫向圓弧轉盤，**可無限循環旋轉**，圖片嚴格限制在展示框內（不溢出）。
- **中央 focus 武器放大到 130%**，兩側武器明顯縮小，營造前大後小的層次。
- **武器之間的間距比目前再加大約 30%**，讓旋轉過渡更順、不擁擠。
- 新增武器時自動跳到最前（維持）。武器照用 `槍枝照片/`（檔名見附錄）。

### 快速套用課程列
- 一排可橫向捲動的課程晶片。
- **滑鼠移到此列上時，滾輪向下＝課程往左捲動**（橫向滾動），不必只靠點擊/箭頭。
- **按鈕按壓特效**：按住時變藍、放開回白，給明確按下回饋。

### 應用情境（小調整）
- **移除「應用情境」標題上方/旁的藍色裝飾圓點**，保持乾淨。

### 方案內容(BOM) 用真實設備照（move here）
- **方案內容(BOM) 卡片改用 `pic/` 資料夾的真實產品照**取代原本 icon 佔位：
  頭盔＝`HTC_VIVE.png`（Focus Vision）；定位設備依所選方式對應
  `Antilatency.png / OptiTrack.png / Ultimate Tracker.png / Vive Tracker.png / Sensoryx.png`
  （Wrist Tracker 暫無圖、用佔位）。檔名對應見 `pic/圖片說明.md`。
- 圖片背景不一（白底/深底），請統一放進**白底圓角卡片**、一致內距，維持 Apple 乾淨感。
- 右側選擇器維持精簡（不放大圖）。武器轉盤的圖另在 `槍枝照片/`。

### 場景示意圖（沿用）
- 場地框 + 人形（取人數級距代表值、靜態不走動）+ 定位設備（AL＝地板燈條、OptiTrack＝桁架環場相機、Vive＝角落基站、免施工＝不畫）。

# 其餘規格（沿用 v3，不變）

- 人數範圍級距 `1 / 2–4 / 5–8 / 9–16 / 17–24 / 25–40`；**滅火器時只給 1 / 2–4**。
- 場地級距小/中/大/超大；定位方式依場地自動禁用（灰度維持現狀）。
- 計價：總價 =（頭盔單價 + 定位每人單價）× 人數代表值 + 定位基礎建置 + Σ(武器開發費)。
- 桌機優先，手機 RWD 之後再優化。
- 純前端單檔、資料集中於一個物件、程式碼乾淨有註解。
