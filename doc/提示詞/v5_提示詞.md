# Claude Design 提示詞 v5 — 模擬訓練解決方案配置器

> 沿用 v4，主要新增「場地↔人數↔定位」交叉約束，並修正圖片路徑接法。
> 使用方式：在現有 v4 成品上續做——開頭加「請以我現有的版本為基礎，套用以下優化」，
> 再貼「===提示詞開始===」到「===提示詞結束===」整段 + 附錄；確認 `槍枝照片/`、`pic/` 已上傳。

## 本版改動摘要（相對 v4）

1. **場地限制可選人數上限**：
   - 小型 → 最多 `5–8`（可選 1 / 2–4 / 5–8）
   - 中型 → 最多 `9–16`（可選 1 / 2–4 / 5–8 / 9–16）
   - 大型 / 超大型 → 不限（可到 25–40）
2. **Antilatency 不支援 25–40 人**：人數選到 25–40 時，Antilatency 自動變灰禁用（最多到 17–24）。
3. **切換場地時自動修正人數**：若目前人數超過新場地的上限，自動降到該場地的最高人數級距。
4. **圖片路徑接對**：武器轉盤用 `uploads/槍枝照片/` 內的圖、方案內容用 `uploads/pic/` 內的圖，
   **照實際檔名接上，不要自己編 `assets/w-*.png` 之類的佔位路徑**（否則下載後會缺圖）。
5. 其餘 v4 規格全部沿用（藍 UI + 紅 logo、左視覺凍結右選單捲動、價格頂部全寬凍結、
   武器轉盤中央放大130%/間距+30%、hover 上浮 highlight、滅火器限 1–4、Apple 大圓角+SF Pro…）。

# 本版重點：場地 ↔ 人數 ↔ 定位 的交叉約束

請把下列規則做成**即時連動的自動篩選**（不相容選項自動變灰禁用並標原因，不讓選到無效組合）：

### 1. 場地決定「可選人數上限」
| 場地 | 可選人數級距 |
|------|-------------|
| 小型 (<36㎡) | 1 / 2–4 / **5–8** |
| 中型 (36–81㎡) | 1 / 2–4 / 5–8 / **9–16** |
| 大型 (81–300㎡) | 1 / 2–4 / 5–8 / 9–16 / 17–24 / 25–40 |
| 超大型 (>300㎡) | 1 / 2–4 / 5–8 / 9–16 / 17–24 / 25–40 |

- 超過上限的人數級距**變灰禁用**。
- **切換場地時**：若目前人數超過新場地上限，自動降到該場地的最高級距。

### 2. 定位方式的人數限制
- **Antilatency 不支援 25–40 人**（最多 17–24）。當人數＝25–40 時，Antilatency 自動變灰禁用、標原因（如「不支援 25–40 人」）。
- 其餘定位方式的場地限制沿用既有規則。

### 3. 武器特例（沿用）
- 選了「滅火器」時，人數只顯示 1 / 2–4。

> 以上多條約束需**同時生效**（取交集）：實際可選人數 = 場地上限 ∩（若選滅火器則 ≤2–4）；
> 實際可選定位 = 場地允許 ∩（人數 25–40 時排除 Antilatency）。

# 圖片路徑（務必接對）

- 武器轉盤：用 `uploads/槍枝照片/` 內的去背 PNG（檔名對應見附錄）。
- 方案內容(BOM)：用 `uploads/pic/` 內的真實設備照（頭盔 + 各定位設備，對應見附錄）。
- **不要自行編造 `assets/w-pistol.png` 等不存在的佔位路徑**；請指向實際上傳的檔案，確保下載完整專案後不缺圖。

# 其餘規格（沿用 v4，不變）

- 藍 UI `#0071e3` + 紅 logo（緋紅 ≈ #D6173E，頂部橫向 lockup）。
- 版面：頂部全寬價格列（凍結）；左欄（武器轉盤＋場景）凍結永遠在；右欄選單捲動。第一屏看到武器＋場地。
- 武器轉盤：去背產品照、無限循環、限制在框內不溢出；中央 focus 放大 130%、兩側縮小、武器間距加大 30%；新增武器跳最前。
- **所有可互動元素 hover 都要微微上浮 + highlight**（最高優先）。
- 快速套用課程：滑鼠滾輪可橫向捲動 + 按鈕按壓特效（按住藍/放開白）。
- Apple 連續曲率大圓角；字體 `-apple-system, "SF Pro Display", "SF Pro Text", "PingFang TC", "Noto Sans TC"`。
- 場景示意圖：場地框＋人形（人數代表值、靜態）＋定位設備（AL 地板燈條 / OptiTrack 桁架 / Vive 角落基站 / 免施工不畫）。
- 應用情境（警勤/軍事/消防）選中態一律藍、不換色；「應用情境」上方不放藍點。
- 計價：總價 =（頭盔單價 + 定位每人單價）× 人數代表值 + 定位基礎建置 + Σ(武器開發費)。
- 桌機優先；純前端單檔、資料集中於一物件、程式碼乾淨有註解。
