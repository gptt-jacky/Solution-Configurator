# Claude Design 提示詞 v2 — 模擬訓練解決方案配置器

> 依「產品配置器3」會議回饋優化。v1 的整體架構保留，本版主要改動見下方摘要。
> 使用方式：複製「===提示詞開始===」到「===提示詞結束===」整段 + 下方「內容資料附錄」貼給 Claude Design。
> 若要在現有版本上續做，可加一句：「請以我現有的版本為基礎，套用以下優化」。

## 本版改動摘要（相對 v1）

1. **武器改成「轉盤 Carousel」**＝畫面主角：已選武器以**去背產品照**橫向圓弧排列，正中那張最大最清晰、左右兩側縮小淡化；可點箭頭/拖曳旋轉。武器是重點。
2. **新增「消防」情境**（共三種：警勤／軍事／消防），三種情境**用不同強調色**區分；切換情境時下方武器清單跟著換。
3. **滑鼠 hover 一律要有 highlight**（微微浮起 + 陰影），這是硬需求（v1 漏做）。
4. **圓角改成 Apple 風（連續曲率 squircle / 大圓角）**，全站統一。
5. **人數改成「範圍級距」**：`1 / 2–4 / 5–8 / 9–16 / 17–24 / 25–40`（最多 40 人）。
6. **第一頁就看到全部重點**：右側配置面板**凍結 sticky**，左側內容捲動；點某區塊可自動捲到對應位置。
7. **場景示意圖加大、別太空**；人形數量反映人數級距（不需走動動畫）；定位設備依方式顯示（Antilatency＝**地板燈條/燈點**、OptiTrack＝桁架環場相機、Vive＝角落基站）。
8. **方案內容（BOM）卡片放大、用自家產品圖**（不要外連他牌產品頁，授權問題）。
9. **定位方式維持簡單、低調呈現**（屬副廠設備，客戶不需懂太多），**不要做轉盤**。

# 角色

你是資深 **產品設計師 + 前端工程師**，擅長 Apple 官網等級的極簡、乾淨、高質感互動網頁。
請設計並實作一個 **互動式產品配置器網站**（汽車線上選配風格），純前端、可直接用瀏覽器開啟。

# 產品背景

一家軍警消 XR 模擬訓練公司（全球動力科技 / BigxReality）的**模擬訓練解決方案配置器**。
使用者是**業務／行銷**，面對政府／軍警消客戶。客戶常「不知道自己要什麼」，只給模糊需求
（場地多大、要練什麼武器、幾個人）。本工具要把模糊需求一步步引導成
**具體方案 + 即時報價 + 場域示意圖**，且**武器/裝備是賣點主角**。

# 最高指導原則（務必嚴格遵守）

1. **視覺像 Apple 官網**：極簡、大量留白、層級清楚、內容優於裝飾。
2. **背景純白** `#ffffff`，次級區塊可用極淺灰 `#f5f5f7`。
3. **圓角用 Apple 風連續曲率（squircle / 平滑大圓角）**，全站統一，不要生硬的小 R 角。
4. **所有可互動元素，滑鼠移過去都要有 hover highlight**：微微上浮（translateY -1~2px）+ 輕陰影 + 底色/邊框變化，平滑過渡 150–200ms。**這是硬需求，不可省略。**
5. **即時反應**：任一選擇變動，武器轉盤、場景示意圖、右側報價、下方 BOM 全部立即更新。
6. **第一頁就看到全部重點**：右側配置面板**凍結（sticky）**不隨捲動跑掉；左側視覺/內容可捲動。
7. **約束式篩選**：選了場地後，不相容的定位方式**自動變灰禁用**並標註原因，不讓使用者選到無效組合。
8. **隱藏技術術語**：對使用者只講白話（場地大小、能不能施工），不要出現 inside-out/outside-in/紅外線。

# 視覺規範

- **配色**
  - 背景 `#ffffff`；次級區塊 `#f5f5f7`
  - 主文字 `#1d1d1f`；次要文字 `#6e6e73`；分隔線 `#d2d2d7`；禁用 `#a1a1a6`
  - **情境強調色（三種情境各一色，用於選中態與該情境的點綴）**：
    - 警勤＝藍 `#0071e3`／軍事＝墨綠 `#3a6b35`／消防＝橘紅 `#e8590c`
  - 主按鈕用「目前情境的強調色」，hover 時加深
- **字體**：`'Noto Sans TC', -apple-system, 'SF Pro', 'PingFang TC', sans-serif`。標題粗體 600–700，內文 400。
- **陰影**：極淡 `0 2px 12px rgba(0,0,0,.06)`，hover 略增。
- **間距**：寬鬆，區塊間 40–72px。
- **動效**：狀態切換 ease 150–250ms；轉盤旋轉與示意圖切換用平滑過渡/淡入。

# 左欄

### ★ 武器轉盤（畫面主角）
- 把**已選武器**以**去背產品照片**橫向**圓弧/卡片轉盤**排列。
- **正中央那張最大、最清晰**；左右兩側的縮小且半透明淡化（往後退的層次感）。
- 提供左右箭頭（與滑鼠拖曳）可**旋轉**切換，中央焦點隨之改變。
- **只顯示已選武器**：在右側清單勾選一項，該武器就以產品照加入轉盤；取消勾選即移出。
- 每張卡：產品照 + 武器名稱 + 開發費（小字）。中央卡可點「移除」。
- 未選任何武器時，轉盤顯示淡淡的提示卡「從右側選擇武器與裝備」。
- 產品照**方向一致**（側面、槍口/朝向統一），維持高級乾淨。
- **武器照片已提供**：使用上傳的 `槍枝照片/` 資料夾內去背 PNG（檔名對應見附錄）。
  尚無照片的品項（警棍、電擊槍、辣椒水罐、榴彈槍、無人機、載具、滅火器、噴水槍）先用乾淨佔位卡。

### 場景示意圖（轉盤下方，次要）
- 白底卡片，畫**場地俯視框 + 人形 + 定位設備**，依選擇即時變化：
  - **場地大小**不同 → 場地框比例變大/變小。
  - **人數**不同 → 框內**人形數量**跟著變（取級距代表值；**不需走動動畫**，靜態排列即可）。
  - **情境**不同 → 用該情境的點綴色 / 氛圍（警勤室內、軍事野外、消防火場）。
  - **定位方式**不同 → 畫出對應設備：Antilatency＝**地板燈條/燈點**；OptiTrack＝上方桁架環場相機；Vive＝角落基站；免施工方式＝不畫額外設備。
- 圖下一行摘要：「軍事模擬 · 8 人 · 大型場地(180㎡) · OptiTrack」。

# 右欄（凍結配置面板）

### ① 快速套用課程
- 一排可橫向捲動的晶片，點一下**一鍵帶入** default 方案（情境/武器/人數/場地/定位全套），之後仍可改。

### ② 應用情境（三選一）
- 三張卡片：**警勤模擬 / 軍事模擬 / 消防模擬**，各含標題 + 一句話描述。
- **三種情境用不同強調色**（藍/綠/橘紅），選中態明顯。
- 切換情境時，**下方武器/裝備清單跟著換**（警勤＝警用；軍事＝軍用；消防＝消防）。

### ③ 武器/裝備（多選清單）
- 標題「武器與裝備（可複選）」。緊湊的多選清單/晶片，依情境顯示對應品項。
- 勾選 → 立即加進左側**武器轉盤**並計入報價。每項顯示名稱（之後加小圖示）。
- 這是和轉盤連動的「選單」；轉盤是「展示」。

### ④ 同時受訓人數（單選，範圍級距）
- 六個範圍按鈕：`1 / 2–4 / 5–8 / 9–16 / 17–24 / 25–40`。
- 計價以該級距代表值估算（如 2–4 取 4、5–8 取 8…）。

### ⑤ 場地大小（單選級距）
- 四個按鈕：`小型(<36㎡) / 中型(36–81㎡) / 大型(81–300㎡) / 超大型(>300㎡)`。
- 選了立即觸發定位方式自動篩選。

### ⑥ 追蹤定位方式（單選，自動禁用，低調呈現）
- 列出定位系統（名稱 + 一句話優點 + 是否需施工標籤）。
- **依場地自動禁用**不適用者（變灰、不可點、標原因如「需大型以上場地」）。
- 視覺上**比武器區低調**（這是副廠設備，客戶不需懂太多）；可選者一樣要有 hover highlight。

### 價格與 CTA
- 「預估價格」大字（32–40px 粗體）即時顯示；旁標「NT$（預估）」。
- 主按鈕「匯出報價」（情境強調色）；次按鈕「加入清單」（白底描邊）。皆有 hover 上浮。

# 下方明細

- **方案內容（BOM）**：**較大**卡片網格，每張＝一個品項 + 數量（頭盔×人數、定位設備×人數、施工建置、各選武器）。**用自家產品圖**（先佔位），不要外連他牌頁面。
- **規格摘要**：條列場地級距、人數、定位方式、是否施工、武器配置。

# 互動行為總結

- 任一選擇變更 → 轉盤/場景/報價/BOM 即時更新；不相容定位自動禁用；不報錯不卡住。
- **全站可互動元素都要 hover 上浮 highlight**。
- 右欄凍結、左欄捲動；點右欄某區塊時左側對應視覺更新。

# 響應式

- 桌面兩欄；平板/手機改單欄堆疊：武器轉盤在上、配置面板其次、明細最後。觸控目標 ≥44px。

# 技術要求

- 純前端單檔（HTML/CSS/JS），不需後端，可直接開啟。
- 資料（情境/武器/級距/定位/價格/課程模板）集中於一個物件，方便改。
- 計價：總價 =（頭盔單價 + 定位每人單價）× 人數代表值 + 定位基礎建置 + Σ(所選武器開發費)。
- 程式碼乾淨、有註解。先產出完整可運作成品，再附簡短設計說明。
