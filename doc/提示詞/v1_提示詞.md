# 角色

你是一位資深 **產品設計師 + 前端工程師**，擅長 Apple 官網等級的極簡、乾淨、高質感網頁設計。
請為我設計並實作一個 **互動式產品配置器網站**（類似汽車線上選配 / OptiTrack「Build My System」）。

# 產品背景

這是一家軍警 XR 模擬訓練公司（BigxReality / 全球動力科技）的**模擬訓練解決方案配置器**。
**使用者是業務／行銷人員**，用它面對政府／軍方客戶。客戶通常「不知道自己要什麼」，
只給得出模糊需求（場地多大、要練什麼武器、幾個人）。這個工具要把模糊需求，
一步步引導成**具體方案 + 即時價格 + 場域視覺示意圖**。

# 最高指導原則（務必嚴格遵守）

1. **視覺像 Apple 官網**：極簡、大量留白、資訊層級清楚、內容優於裝飾。
2. **背景純白**（`#ffffff`），區塊可用極淺灰（`#f5f5f7`）分層。整體乾淨、無雜訊。
3. **所有可互動元素，滑鼠移過去都要有 hover highlight**（按鈕、卡片、選項、連結、清單項目皆是）。
   hover 要有平滑過渡（150–200ms），可用淺灰底、邊框變色、輕微上浮或陰影。
4. **即時反應**：任何選擇變動，右側價格、下方 BOM 清單、左側示意圖都要**立即更新**。
5. **約束式篩選（核心）**：當使用者選了場地/人數，**不相容的定位方式要自動變灰禁用**
   （不可選，並顯示為什麼不適用的簡短說明），不要讓使用者選到無效組合。
6. **隱藏技術術語**：對使用者只講白話（場地大小、能不能施工），
   不要出現 inside-out / outside-in / 紅外線 等術語。

# 視覺規範

- **配色**
  - 背景：`#ffffff`；次級區塊：`#f5f5f7`
  - 主文字：`#1d1d1f`；次要文字：`#6e6e73`
  - 主色（強調 / 選中 / 主按鈕）：`#0071e3`（藍）；hover 加深 `#0077ed`
  - 分隔線：`#d2d2d7`；禁用：文字 `#a1a1a6`、底 `#f5f5f7`
  - （主色之後可換成品牌色，先用此藍）
- **字體**：無襯線，優先 `-apple-system, "SF Pro", "Noto Sans TC", "PingFang TC", sans-serif`。
  標題粗體（600–700），內文 400。中文字距舒適。
- **圓角**：卡片 / 按鈕 12–16px，選項小卡 10px。
- **陰影**：極淡（如 `0 2px 12px rgba(0,0,0,0.06)`），hover 時略增。
- **間距**：寬鬆，區塊間 48–80px，元素間 16–24px。
- **動效**：所有狀態切換用 ease 過渡 150–250ms；示意圖切換可加淡入。

# 版面結構（桌面）

頂部：簡潔導覽列（白底、左 Logo、置中標題「方案配置器」、右側「重置 / 匯出報價」）。

主體為**兩欄**（左大右小，約 6:4），右欄 sticky 跟隨捲動：

```
┌───────────────────────────────┬────────────────────────────┐
│ 左欄：場域視覺示意圖            │ 右欄：配置面板（sticky）     │
│  - 大圖卡，白底，置中           │  ① 應用情境（情境卡片）       │
│  - 依「情境 + 場地 + 人數 +     │  ② 武器/裝備（多選晶片）      │
│    武器」即時變化的場景         │  ③ 人數（級距按鈕）           │
│  - 圖下方一行：目前配置摘要     │  ④ 場地大小（級距按鈕）       │
│                               │  ⑤ 定位方式（卡片，自動禁用） │
│                               │  ───────────────────────    │
│                               │  預估價格（大字）             │
│                               │  [匯出報價]   [加入清單]      │
└───────────────────────────────┴────────────────────────────┘
┌──────────────────────────────────────────────────────────────┐
│ 下方明細區（白底分區）                                          │
│  ・方案內容 BOM（卡片網格：頭盔×N、定位設備×N、施工、各武器）   │
│  ・規格摘要（場地級距、人數、定位方式、是否需施工）            │
└──────────────────────────────────────────────────────────────┘
```

# 右欄各區塊規格（逐一說明）

### ① 應用情境（單選）
- 兩張並排卡片：**警勤模擬 / 軍事模擬**，各含標題 + 一句話描述。
- 預設選「警勤模擬」。選中：藍框 + 淺藍底；hover：淺灰底 + 輕陰影。
- 切換情境時，下方**武器清單要跟著換**（警勤只顯示警用裝備，軍事顯示軍用武器）。

### ② 武器/裝備（多選，重頭戲）
- 標題「武器與裝備（可複選）」。以**晶片 / 小卡片網格**呈現，可多選。
- 依情境分組顯示（例：警勤＝手槍/警棍/電擊槍/辣椒水罐；軍事＝步槍/機槍/狙擊/飛彈…）。
- 每張卡：名稱（之後可放小圖示）。選中：藍框+勾；hover：highlight。
- 這是重點區，視覺上要好按、好瀏覽。

### ③ 人數（單選級距按鈕）
- 一排圓角按鈕：`1 / 2 / 4 / 6 / 8 / 12 / 16 / 24`。
- 選中：實心藍底白字；hover：淺灰底。

### ④ 場地大小（單選級距按鈕）
- 四個按鈕：`小型(<36㎡) / 中型(36–81㎡) / 大型(81–300㎡) / 超大型(>300㎡)`。
- 選中態同上。**選了之後立刻觸發定位方式的自動篩選**。

### ⑤ 定位方式（單選卡片，自動禁用不相容）
- 卡片列出可選的定位系統（名稱 + 一句話優點 + 是否需施工標籤）。
- **依場地級距自動禁用**：不適用者變灰、不可點，並標註原因（如「不適用大型場地」）。
- 可選者 hover highlight；選中藍框。預設自動選中第一個可用的。

### 價格與 CTA
- 「預估價格」用大字（28–40px，粗體）即時顯示總價，旁標「NT$（預估）」。
- 主按鈕「匯出報價」（實心藍）；次按鈕「加入清單」（白底深字描邊）。兩者皆有 hover。

# 左欄：場域視覺示意圖

- 白底大卡片，置中一張**會隨選擇變化的示意圖**（先用 SVG / 向量插畫即可）：
  - **情境**不同 → 場景氛圍不同（警勤＝室內訓練場；軍事＝戰場/野外）。
  - **人數**不同 → 圖中人形數量跟著變。
  - **場地大小**不同 → 場地框框比例跟著變大變小。
  - **武器**不同 → 人形手上裝備示意（有就好，不需精緻）。
  - **定位方式**不同 → 周邊設備示意（需施工者畫出相機/基站/地墊；免施工者不畫）。
- 圖下一行小字摘要：「軍事模擬・8 人・大型場地(180㎡)・OptiTrack」。
- 切換時示意圖用淡入淡出過渡。

# 下方明細區

- **方案內容（BOM）**：卡片網格，每張一個品項 + 數量（頭盔×人數、定位設備×人數、施工建置、各選武器）。
- **規格摘要**：條列場地級距、人數、定位方式、是否需施工、追蹤配置。

# 互動行為總結

- 任一選擇變更 → 即時重算價格、重繪 BOM、更新左側示意圖、重新計算定位方式可選性。
- 不相容組合自動禁用，不報錯、不卡住。
- 全站可互動元素都有 hover highlight 與平滑過渡。

# 響應式（RWD）

- 桌面兩欄；平板與手機改為**單欄堆疊**：示意圖在上、配置面板在下、明細最後。
- 觸控目標夠大（≥44px），手機上級距按鈕可換行。

# 技術要求

- **純前端**，單一 `index.html` + CSS + JS（可用原生或輕量框架），**不需後端**，可直接用瀏覽器開啟。
- 資料（情境/武器/級距/定位方式/價格）請集中在一個 JS 物件或 JSON，方便日後修改。
- 價格計算邏輯：總價 =（頭盔單價 + 定位每人單價）× 人數 + 定位基礎建置 + Σ(所選武器開發費)。
- 程式碼乾淨、有註解、易維護。

請先產出完整可運作的成品，再附上簡短的設計說明（配色、字體、互動重點）。
