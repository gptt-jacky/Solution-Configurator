# Claude Design 提示詞 v3 — 模擬訓練解決方案配置器

> 依「產品配置器4」會議回饋優化。整體沿用 v2，本版主要改動見摘要。
> 使用方式：在現有 v2 成品上續做最佳——開頭加「請以我現有的版本為基礎，套用以下優化」，
> 再貼「===提示詞開始===」到「===提示詞結束===」整段 + 下方附錄；並確認 `槍枝照片/` 已上傳。

## 本版改動摘要（相對 v2）

1. **顏色全站統一藍色** `#0071e3`：**取消情境換色**（不要軍事綠/消防橘紅）。右上按鈕、主按鈕、價格永遠藍。
2. **價格置頂凍結**：頁面最上方一條**全寬價格列**（像 Apple 購買列），隨時顯示預估價 + 匯出報價，捲動不消失。
3. **凍結方向反轉**：**左側（武器轉盤 + 場景示意圖）凍結永遠在**；捲動時**只有右側選單會動**。
4. **封面一屏看完重點**：第一屏右側選單就要同時看到**武器（在上）＋場地**，面板加寬、排版緊湊，盡量免捲動即可選到這兩項。
5. **武器在場地之上**（武器是重點，先選武器）。
6. **R 角再加大**：更明顯的 Apple 連續曲率（squircle）大圓角，不要還看得到銳角。
7. **字體比照 Apple**：拉丁字母與數字用 Apple 系統字（`-apple-system, "SF Pro Display", "SF Pro Text"`），中文用 `"PingFang TC", "Noto Sans TC"`。
8. **武器轉盤：無限循環 + 不超出展示框**：圖片限制在框內（不要過大、不溢出、不卡頓），可一直旋轉。
9. **滅火器限 1–4 人**：選了滅火器時，人數只給 `1 / 2–4`（若原本選更多自動降到 2–4）。
10. 灰色「無法使用」態**維持現狀**（不用加深）。
11. **桌機優先**，手機 RWD 之後再優化（先確保桌機完美）。

# 角色

你是資深 **產品設計師 + 前端工程師**，擅長 Apple 官網等級的極簡、乾淨、高質感互動網頁。
請在我現有版本基礎上，做一個**互動式產品配置器網站**（汽車線上選配風格），純前端、可直接用瀏覽器開。

# 產品背景

軍警消 XR 模擬訓練公司（全球動力科技 / BigxReality）的**模擬訓練解決方案配置器**，給**業務／行銷**用。
客戶常「不知道自己要什麼」，本工具把模糊需求引導成**具體方案 + 即時報價 + 場域示意圖**，**武器/裝備是賣點主角**。

# 最高指導原則（務必嚴格遵守）

1. **視覺像 Apple 官網**：極簡、大量留白、層級清楚、內容優於裝飾。
2. **背景純白** `#ffffff`，次級區塊 `#f5f5f7`。
3. **全站顏色統一藍** `#0071e3`（選中態、主按鈕、價格、點綴一律藍）；**情境切換不換色**。
4. **圓角用明顯的 Apple 連續曲率（squircle）大圓角**，全站統一，不可看到生硬銳角。
5. **字體比照 Apple**：`font-family: -apple-system, "SF Pro Display", "SF Pro Text", "PingFang TC", "Noto Sans TC", sans-serif;`（拉丁/數字走 SF、中文走蘋方/思源）。
6. **所有可互動元素 hover 都要 highlight**：微微上浮 + 輕陰影 + 底色/邊框變化，過渡 150–200ms。
7. **即時反應**：任一選擇變動 → 轉盤、場景、價格、BOM 立即更新。
8. **約束式篩選**：選場地後不相容定位方式自動變灰禁用並標原因；不讓選到無效組合。
9. **隱藏技術術語**：只講白話（場地大小、能不能施工），不出現 inside-out/outside-in/紅外線。

- **捲動行為**：滾輪往下時，**左欄（武器轉盤＋場景）完全不動**，只有右欄選單捲動。頂部價格列也凍結。
- **第一屏**：右欄要讓**武器（②）與場地（③）都看得到**，不用捲動就能選這兩項（面板加寬、間距收斂）。

# 頂部全寬價格列（新增，凍結）

- 橫跨整頁寬度，sticky 固定在最上方，白底 + 底線。
- 左側：Logo「全球動力科技 · XR 模擬訓練」+「方案配置器」。
- 右側：**預估價格大字**（28–34px 粗體，永遠藍）+「匯出報價」（實心藍按鈕）+「加入清單」（白底描邊）+「重置」。
- **不論情境（警勤/軍事/消防）一律藍色**，不換色。

# 左欄（凍結，永遠在）

### ★ 武器轉盤（主角）
- 把**已選武器**以**去背產品照**做橫向圓弧轉盤：正中最大最清晰、左右兩側縮小且半透明淡化。
- **可無限循環旋轉**（左右箭頭 + 滑鼠拖曳）；**圖片嚴格限制在展示框內**（`object-fit:contain`、設 max 尺寸、`overflow:hidden`），不要過大、不溢出、不造成卡頓。
- 只顯示已選武器；右側勾選 → 加入轉盤、取消 → 移除。未選時顯示淡提示卡。
- 每張卡：產品照 + 名稱 + 開發費（小字）。武器照用 `槍枝照片/`（檔名對應見附錄）；無照片者用乾淨佔位卡。

### 場景示意圖（轉盤下方）
- 白底卡片，畫場地俯視框 + 人形 + 定位設備，即時變化：
  - 場地大小 → 框比例變化；人數 → 人形數量（取級距代表值，**靜態、不走動**）。
  - 定位方式 → Antilatency＝地板燈條/燈點；OptiTrack＝上方桁架環場相機；Vive＝角落基站；免施工＝不畫額外設備。
- 圖下一行摘要：「軍事模擬 · 8 人 · 大型場地(180㎡) · OptiTrack」。

# 右欄（捲動選單）

### 快速套用課程
- 橫向晶片列，點一下一鍵帶入 default 方案（之後可改）。

### ① 應用情境（三選一）
- 三張卡片：**警勤模擬 / 軍事模擬 / 消防模擬**（+ 一句話描述）。**選中態統一藍色，不換色。**
- 切換情境時下方武器清單跟著換（警勤＝警用；軍事＝軍用；消防＝消防）。

### ② 武器與裝備（多選，★放在場地之上）
- 緊湊多選清單/晶片，依情境顯示；勾選即進左側轉盤並計價。

### ③ 場地大小（單選級距）
- `小型(<36㎡) / 中型(36–81㎡) / 大型(81–300㎡) / 超大型(>300㎡)`；選了即觸發定位方式自動篩選。

### ④ 同時受訓人數（單選，範圍級距）
- `1 / 2–4 / 5–8 / 9–16 / 17–24 / 25–40`，計價取代表值。
- **特例：選了「滅火器」時，人數只顯示 `1 / 2–4`**（若原為更多，自動降到 2–4）。

### ⑤ 追蹤定位方式（單選，自動禁用，低調）
- 列出定位系統（名稱 + 一句優點 + 是否施工標籤）；依場地自動禁用不適用者（變灰標原因，灰度維持現狀）。
- 視覺上比武器低調（副廠設備），但可選者一樣要有 hover highlight。

# 下方明細

- **方案內容（BOM）**：較大卡片網格，自家產品圖（先佔位，不外連他牌）；含頭盔×人數、定位設備×人數、施工建置、各選武器。
- **規格摘要**：場地/人數/定位/是否施工/武器配置。

# 互動行為總結

- 任一變更 → 轉盤/場景/價格/BOM 即時更新；不相容定位自動禁用；不報錯不卡住。
- 全站可互動元素 hover 上浮 highlight。
- 頂部價格列凍結、左欄凍結、右欄捲動。

# 響應式

- **桌機優先、做到完美**；手機先單欄堆疊可用即可（之後再專門優化）。

# 技術要求

- 純前端單檔（HTML/CSS/JS），可直接開啟。資料集中於一個物件方便改。
- 計價：總價 =（頭盔單價 + 定位每人單價）× 人數代表值 + 定位基礎建置 + Σ(武器開發費)。
- 程式碼乾淨、有註解。
