## 本版改動摘要（相對 v6）

1. **補 Wrist Tracker 設備圖**：方案內容用 `uploads/pic/Wrist Tracker.png`，不再用佔位。
2. **定位改「推薦 + 備選」**：不只給一個最優解；標出**建議方案（推薦）**，並把**其他符合條件的方案也一併列出**（可點選切換，切換時報價/BOM/場景跟著更新）。
3. **Wrist Tracker 人數上限**：只用於 **1～4 人**（5–8 起不使用）。
4. **武器數量可調整**（每把武器加 +/− 數量，可手動改）。預設：
   - **手槍 / 步槍 / 警棍 / 電擊槍 / 辣椒水罐 = 人數**（隨人數連動，除非手動改過）
   - 其他武器預設 **1 把**
5. **價格模型 B + 量大折扣**（每種武器各自依數量累計）：
   - 第 1 把原價 → 第 2–5 把 7 折 → 第 6–15 把 5 折 → 第 16 把以上 3 折
6. **價格字體縮小**：頂部價格列字級調小，**手機上保持一行**（不要折成兩行）。
7. **場景示意圖隨「現場條件」變化 + 動畫**：不同勾選對應不同場景（現在都一樣，要修）。
8. 其餘 v6 規格沿用（藍 UI+紅 logo、左視覺凍結右選單捲動、價格頂部凍結、武器轉盤、hover 上浮、Apple 圓角+SF Pro、場地↔人數上限、滅火器限1–4、現場條件篩選、前臺不強調廠牌…）。

# 角色

你是資深 **產品設計師 + 前端工程師**，擅長 Apple 官網等級的極簡乾淨互動網頁。
請在我現有版本基礎上，繼續打磨這個**模擬訓練解決方案配置器**（純前端、可直接開啟）。

# 本版重點

## 1. 武器數量可調整 + 量大折扣

- 每把已選武器在右側顯示**數量 +/− 控制**（可手動輸入/增減）。
- **預設數量**：
  - 手槍 / 步槍 / 警棍 / 電擊槍 / 辣椒水罐 → **＝目前人數代表值**（隨人數連動；使用者手動改過後就不再自動覆蓋）。
  - 其他武器（衝鋒槍/機槍/狙擊/榴彈/飛彈/無人機/載具/滅火器/噴水槍） → **預設 1 把**。
- **價格模型 B（開發費一次性 + 每把製作費，用折扣近似）**：每種武器依其數量逐把計價：
  - 第 1 把：原價（100%）
  - 第 2–5 把：7 折（70%）
  - 第 6–15 把：5 折（50%）
  - 第 16 把以上：3 折（30%）
  - 該武器小計 = 各把價格加總；總價 = Σ各武器小計 +（頭盔單價 + 定位每人單價）×人數 + 定位基礎建置。

## 2. 定位方案：推薦 + 備選

- 依「場地 + 人數 + 現場條件」篩出**所有符合的定位方案**。
- 標出**建議方案**（成本最低者，加「推薦」標籤），並**把其他符合的方案也列出**（依成本由低到高）。
- 使用者**可點選切換**到備選方案；切換時報價、BOM、場景示意圖同步更新。
- 仍以白話/設備為主，前臺不強調定位廠牌名稱。
- **Wrist Tracker 只在 1～4 人時納入**（5–8 人起不出現）。
- 若某條件把方案全排除，顯示「需技術單位評估」提示，不報錯。

## 3. 場景示意圖隨現場條件變化（含動畫）

左側場景示意圖要**依勾選的現場條件改變**（目前都一樣，需修正），切換時用平滑過渡/淡入當動畫：
| 現場條件 | 場景表現 |
|---------|---------|
| 室外 | 戶外背景（天空/草地） |
| 強光（窗多/日照） | 明亮、有窗/陽光感 |
| 照明不足 | 偏暗場景 |
| 可鋪地墊 | 地面顯示地墊/燈條 |
| 可架桁架 | 顯示上方桁架 + 相機 |
| 需移動/快速佈置 | 簡易可攜設備、少基礎建設 |

- 同時仍反映：場地大小（框比例）、人數（人形數，靜態）、最終定位設備。

## 4. 價格字體

- 頂部全寬價格列的價格字級**調小**，確保**手機（窄螢幕）上維持一行**、不折行。

# 其餘規格（沿用 v6，不變）

- 藍 UI `#0071e3` + 紅 logo（頂部橫向 lockup，`uploads/pic/LOGO.png`）。
- 版面：頂部價格列凍結；左欄（武器轉盤＋場景示意圖）凍結永遠在；右欄選單捲動。
- 右欄順序：快速套用 → 應用情境（警勤/軍事/消防，選中藍、不換色）→ 武器與裝備（多選＋數量）→ 場地大小 → 人數 → 現場條件 → 定位結果（推薦+備選）。
- **場地↔人數上限**：小型≤5–8、中型≤9–16、大型/超大型不限；切換場地超上限自動下修。
- **滅火器**：人數限 1–4。
- 武器轉盤：去背產品照（`uploads/槍枝照片/`）、無限循環、限框內不溢出、中央放大130%、間距+30%、新增跳最前。
- **方案內容(BOM)**：用 `uploads/pic/` 真實設備照（頭盔 `HTC_VIVE.png` + 定位設備，含 `Wrist Tracker.png`），白底圓角卡片、不強調廠牌；數量反映武器數量。
- 快速套用課程：滑鼠滾輪可左右橫向捲動 + 按鈕按壓特效。
- **所有可互動元素 hover 都要上浮 + highlight**（最高優先）。
- Apple 連續曲率大圓角；字體 `-apple-system, "SF Pro Display", "SF Pro Text", "PingFang TC", "Noto Sans TC"`。
- 應用情境上方不放藍點；手槍維持多重區分（電動/瓦斯）。
- 圖片路徑指向 `uploads/`，不要自編 `assets/w-*.png` 佔位。
- 桌機優先（但價格列要顧到手機一行）；純前端單檔、資料集中於一物件、程式碼乾淨有註解。
