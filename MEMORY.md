# MEMORY — 解決方案配置器

> 跨 session 穩定知識。按主題分類，保持精簡。

## 專案概述

- **名稱**：解決方案配置器（**模擬訓練解決方案配置器**）
- **公司**：**GLOBAL POWER TECHNOLOGIES CO., LTD.（全球動力科技）**，AI XR 品牌 = BigxReality，官網 [https://www.bigxreality.com/tw/home](https://www.bigxreality.com/tw/home)
- **官網三大解決方案**：警勤模擬 / 軍事模擬（五大模組）/ 智慧城市（數位雙生）。詳見 `doc/官網解決方案架構.md`
- **MVP 範圍**：頂層情境 = **警勤 + 軍事 + 消防**（智慧城市排除；消防於產品配置器3會議追加）；**人數、場地大小列為明確配置選項**（範圍差很大，且是定位方式自動排除的依據）
- **配置流程**：情境 → 武器(多選) → 人數/場地 → 定位方式(自動排除) → BOM/價格/示意圖
- **武器（重頭戲）**：手槍(電動/瓦斯)/步槍/衝鋒槍/機槍/狙擊/飛彈火箭榴彈/警棍/電擊/辣椒水/載具(悍馬車/快艇/戰車/無人機)。詳見 `doc/武器與訓練課程清單.md`
- ⚠️ 官網為 JS SPA，WebFetch 抓不到內文；型錄主要靠 Excel + 使用者貼上的官網文字
- **使用者**：**業務／行銷**用（非終端消費者；面對政府/軍方客戶）
- **目標**：把客戶模糊需求（場地/武器/人數）→ 推薦方案 + 價格 + 示意圖；業務以前接不起來這種前期諮詢
- **核心難點**：**約束式篩選**——選某選項自動排除不相容選項（各定位系統限制多、彼此衝突）
- **設計原則**：隱藏技術術語（inside-out/outside-in 不給看），用白話引導（場地越大→…、定位東西越多→…）
- **互動體驗**：**汽車配置器式**——選項變動時左側場域模擬示意圖即時變化（核心賣點）
- **UX 標竿**：OptiTrack「Build My System」(結構與資料模型幾乎一對一，詳見 `doc/參考網站.md`)
- **技術形式**：純前端網頁（資料 JSON 驅動，可直接部署）
- **MVP 態度**：先做骨架 + 假資料，價格先粗估，再逐步填真資料
- **設計**：之後用 Claude Design 產生，需對齊官網科技感／極簡專業風格
- **需求細節**：見 `doc/需求_會議紀要.md`

## 核心資料模型（詳見 DATA_MODEL.md）

- **6 種定位方式**：Antilatency(AL)、Optitrack、Ultimate Tracker(UT)、Sensoryx(SX)、Wrist Tracker(WT)、Vive tracker
- **5 種追蹤配置**：頭 / 頭+雙手 / 頭+雙手+腰 / 頭+雙手+腰+雙腳 / 手持設備
- **輸入**：定位方式 + 追蹤配置 + 人數 + 場域大小 → **輸出**：BOM 明細 + 成本
- 主表「定位方式場域人數成本總表」是 6×5 成本矩陣；其餘 45 張為客戶實際案例（可作驗證樣本）
