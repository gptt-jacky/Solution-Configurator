# 配置器資料模型

> 來源：`XR解決方案成本試算.xlsx` 主表「定位方式場域人數成本總表」。
> 本檔定義配置器的輸入、輸出與計算邏輯，作為前端 `data.json` 與計算引擎的設計依據。

---

## 1. 核心結構：6 × 5 成本矩陣

使用者選定一種**定位方式**與一種**追蹤配置**，搭配**人數**與**場域大小**，得到該組合的 BOM 與成本。

### 定位方式（6 種）與限制

| 代號 | 定位方式 | 主要限制 / 備註 |
|------|---------|----------------|
| AL | Antilatency | 場域大小需 > 10 m² |
| Opti | Optitrack | 建議 > 81 m²；已扣除頭盔價格；其他限制見 Optitrack 場域建議配置 |
| UT | Ultimate Tracker | 建議 < 81 m²；每學員（全身含手持）最多 5 顆 UT；已扣頭盔價；LBE mode 尚未取得 HTC 報價 |
| SX | Sensoryx | 1 個 SX hub 最多 20 個 Spatial Tag；Tag 續航 3.5 小時；定位範圍最遠 5 m |
| WT | Wrist Tracker | 建議 < 81 m²；每學員（全身含手持）最多 2 顆 WT；已扣頭盔價 |
| Vive | Vive tracker | 建議 < 36 m²；單一空間最多定位 9 顆 Vive tracker |

> 「為求各方案公平比價，已扣除頭盔價格」適用於 Opti / UT / WT。

### 追蹤配置（5 種，依人體部位）

| # | 配置 | 說明 |
|---|------|------|
| 1 | 頭 | 僅頭部追蹤 |
| 2 | 頭+雙手 | 頭 + 雙手 |
| 3 | 頭+雙手+腰 | 頭 + 雙手 + 腰 |
| 4 | 頭+雙手+腰+雙腳 | 全身 |
| 5 | 手持設備 | 手持武器/設備定位（與身體追蹤分開計算） |

---

## 2. 輸入參數

| 參數 | 型別 | 說明 |
|------|------|------|
| `scenario` | enum | **應用情境：警勤 / 軍事**（MVP 兩種；智慧城市排除） |
| `weapons` | array | **武器/裝備選配（多選，重頭戲）**，見 `武器與訓練課程清單.md` |
| `headcount` | number | **人數（學員數）**——明確配置選項，範圍差很大（1～24+） |
| `fieldSize` | number | **場域大小 (m²)**——明確配置選項，範圍差很大（~10～900） |
| `config` | enum | 追蹤配置（頭/+雙手/+腰/+雙腳/手持，5 種之一） |
| `method` | enum | 定位方式（AL/Opti/UT/SX/WT/Vive）——多依場地/人數**自動排除**後選擇 |

> 黃色儲存格 = 可在備註限制範圍內編輯（即上述輸入）。
> 輸入次序（建議 UI 流程）：情境 → 武器 → 人數/場地 → 定位方式（自動排除）→ 輸出 BOM/價格/示意圖。

---

## 3. 輸出：BOM 與成本

每個組合輸出一張 BOM 表，欄位固定為：

| 欄位 | 說明 |
|------|------|
| 項目 | 主要設備類別（如 AL Alt、AL socket、地墊燈條、攝影機與周邊…） |
| 子項 | 細分（頭/雙手/腰/雙腳/手持設備/備品/AP…） |
| 數量 | 單項數量 |
| 數量總計 | 含備品係數後的總數量 |
| 金額小計 | 該項金額 |
| 金額總計 | 該方案合計（含研發費用轉換率等加成） |

### 各定位方式的典型 BOM 組成

| 定位方式 | 主要品項 |
|---------|---------|
| AL | AL Alt、AL socket、AP、地墊燈條、備品 |
| Opti | 攝影機與周邊 (Slimx13)、puck 定位器、Truss 架含安裝 |
| UT | Focus Vision（頭盔，比價時歸零）、UT（各部位）、勤務設備/武器 |
| SX | SX hub、Spatial Tag、勤務設備/武器 |
| WT | Focus Vision、WT、勤務設備/武器 |
| Vive | Base station、Vive tracker（各部位）、勤務設備/武器 |

## 4. 計算邏輯重點（待精確化）

- **數量總計** 通常 = 數量 × (1 + 備品係數)；備品係數依品項不同（如 AL Alt 備品 0.2 / 0.3）
- **地墊燈條** 數量 ≈ 場域大小 / 10 m²（單位 `10m^2`）
- **金額總計** 在金額小計之外，疊加「研發費用轉換率」等加成（見工作表「研發費用轉換率」）
- **手持設備** 配置獨立於身體追蹤，依武器/勤務設備數量計算

> ⚠️ TODO（待補）：各品項**單價**尚未抽取，分散在產品線工作表中：
> `產品SUT`、`產品GPL_beta/V1/V2`、`產品GPS菲律賓四人雙槍版`、`產品電擊器&手步槍`、
> `Sensoryx採購`、`研發費用轉換率`。建模下一步需從這些表整理出單價表。

## 5b. 定位系統差異 → 約束式篩選依據

各定位系統的**場地布置限制**與**佔地面積限制**差異，是配置器約束式篩選的核心依據，
已整理於 [doc/定位系統比較.md](doc/定位系統比較.md)。兩條可立即落地的篩選邏輯：

- **能否施工**：不能施工 → 只剩 Ultimate Tracker / Wrist Tracker / Sensoryx（免布置）
- **場地大小**：>81m²→AL/OptiTrack；36–81m²→UT/WT/Sensoryx；<36m²→含 Vive；<10m²→排除 AL

> 分類簡化：**Sensoryx 視同 Wrist Tracker**（差別僅體積更小），篩選時歸同群。

## 6. 待釐清問題（下次與使用者對齊）

1. 配置器是否需呈現**單價明細**，或只給方案總價與比較？
2. 「研發費用轉換率」加成是否要對外顯示，或只算內部成本？
3. 手持設備/武器清單是否要做成可選清單（如選 M4A1、刺針飛彈…）？
4. 對外展示是否要遮蔽實際金額（改為「洽詢報價」）？
